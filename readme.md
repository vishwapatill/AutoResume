# AutoresumePDF
 
> Scrape → Parse → Tag → Generate. Your resume, always current, always tailored.
 
AutoresumePDF pulls your latest profile data from LinkedIn and GitHub, uses an LLM to clean, structure, and tag everything, then generates role-tailored resumes and cover letters as polished PDFs — on demand, in seconds.
 
---
 
## How it works
 
```
LinkedIn Export ZIP          GitHub REST API
       │                           │
       └──────────┬────────────────┘
                  ▼
           [ Scraper Layer ]
       Pulls profile, experience,
       repos, READMEs, certifications
                  │
                  ▼
           [ LLM Parser ]
       Combines sources, removes
       duplicates, fills schema fields,
       tags projects by role/tech
                  │
                  ▼
         [ Structured Data ]
       data/summary.txt
       data/projects.json
       data/tags.json
       data/experience.txt
       data/achievements.txt
       data/certifications.txt
                  │
                  ▼
        [ Resume Generator ]
    python create_resume.py \
        --jd job_description.txt \
        --cover
                  │
                  ▼
       Resume_2026-04-17.pdf
         CoverLetter.txt
```

## Pipeline — step by step
 
### 1. Scrape LinkedIn
Reads your official LinkedIn data export ZIP (Settings → Data Privacy → Get a copy of your data). Pulls experience, education, certifications, skills, projects, honors, recommendations, volunteer work, and languages. Detects what changed since the last run so only stale sections are re-processed.
 
### 2. Scrape GitHub
Hits the GitHub REST API via PyGithub. Pulls your public repos (non-forked, non-archived), README snippets, language byte counts, star counts, and topics. Caches results locally — re-fetches only when data is older than 24 hours.
 
### 3. LLM combines and structures everything
An LLM (OpenAI, Claude, or local Ollama) receives both data sources and maps them into predefined schemas:
 
```
data/exp/schema.json        →  experience.txt
data/projects/schema.json   →  projects.json
```
 
It resolves conflicts between sources (e.g. job title in LinkedIn vs GitHub bio), removes redundant entries, merges LinkedIn projects with GitHub repos where they refer to the same thing, and writes clean, concise bullet points in a consistent voice.
 
### 4. Redundant data is removed
Duplicate entries across LinkedIn and GitHub are deduplicated by the LLM before anything is saved. If a project appears in both LinkedIn projects and GitHub repos, it gets one canonical entry with the richer metadata.
 
### 5. Projects are tagged
The LLM reads each project's description, README snippet, language, and topics and assigns role tags from a fixed taxonomy:
 
```
fullstack  |  datascience  |  automation  |  sde  |  devops  |  ml
```
 
Tags are saved to `data/tags.json` as `{ "tag": ["proj_id", ...] }` so the resume generator can quickly filter relevant projects for any role.
 
### 6. Processed data is saved
All structured output lands in `data/`:
 
| File | Contents |
|---|---|
| `summary.txt` | 3–5 sentence professional summary generated by LLM |
| `projects.json` | All projects with id, name, description, tags, tech stack, urls |
| `tags.json` | Inverted index: tag → list of project ids |
| `experience.txt` | Work history, one entry per role with bullet points |
| `achievements.txt` | Quantified wins extracted from experience descriptions |
| `certifications.txt` | Certification name, issuer, date, license number |
 
### 7. Generate a resume
```bash
python create_resume.py --jd job_description.txt --cover
```
 
Pass a job description file and the tool selects the most relevant projects (via tag matching + LLM ranking), picks the right experience bullets, and populates a pre-built HTML template.
 
### 8. HTML template is populated
Role-specific HTML/CSS templates live in `pdf/templates/`. The selected data is rendered into the template via Jinja2. You can preview the output by opening the HTML file in a browser before committing to PDF.
 
### 9. PDF and cover letter are generated
Playwright launches a headless Chromium instance, renders the HTML, and prints it to a versioned PDF:
 
```
Resume_2026-04-17.pdf
CoverLetter.txt
```
